# 내용-구조 연산을 위한 유일한 아키텍처로서의 메모리 내 처리:
# Hadamard 연산자에서 ReRAM 크로스바 회로까지

권혁준 — 2026년 4월

---

**초록.** Hadamard 곱 T = C⊙S — 그래프 위에서 내용과 구조의 유일한 변분 안정 합성으로 증명된(Kwon 2026a) — 는 직접적 하드웨어 실현을 갖는다: 아날로그 크로스바 배열. 저항성 메모리(ReRAM) 크로스바에서 옴의 법칙은 메모리와 프로세서 사이의 데이터 전송 없이 각 접합에서 입력 전압(내용)과 전도도(구조)의 원소별 곱을 계산한다. 이 대응이 유비가 아닌 구조적 동일성임을 보인다: (1) 크로스바는 행렬 곱이 아닌 Hadamard 곱을 자연스럽게 계산한다; (2) 자기수반 정규화 D⁻¹/²AD⁻¹/²가 대칭 전도도 스케일링에 사상되어 차수 불균형을 자동 보정한다; (3) 계산 복잡도가 O(n²)(행렬 곱)에서 O(|E|)(에지 수)로 감소하여 실제 그래프의 희소성과 일치한다; (4) 변분 안정성 조건(α = β = 1)이 모든 비선형 메모리 내 활성화 방식을 배제한다. 이 결과들은 메모리 내 처리를 공학적 최적화가 아닌, 내용-구조 연산을 위한 물리적으로 필연적인 아키텍처로 확립한다.

---

## 1. 대응

### 1.1 문제

그래프 신경망, 추천 시스템, 지식 그래프, 격자 시뮬레이션 모두 "노드의 내용과 연결의 구조를 결합하라"는 형태의 연산을 수행한다. 표준 접근은 저장(메모리)과 계산(프로세서)을 분리한다: 구조는 메모리에 저장되고, 내용은 프로세서로 전송되며, 계산이 일어나고, 결과가 다시 전송된다.

이 분리가 폰 노이만 병목을 만든다: 데이터 전송이 에너지와 시간을 지배한다. 수백만 노드의 희소 그래프에서 전송 비용이 계산 비용을 압도한다.

### 1.2 관찰

논문 A는 내용 C와 구조 S의 유일한 변분 안정 합성이 Hadamard(원소별) 곱임을 증명했다: T_{ab} = C_{ab} · S_{ab}. 이 연산은 특별한 성질을 갖는다: **국소적이다.** 각 출력 원소가 같은 위치의 대응 입력 원소에만 의존한다. 어떤 원소도 다른 위치에 의존하지 않는다.

ReRAM 크로스바 접합은 전도도 G_{ab}(구조)를 저장하고 입력 전압 V_a(내용)를 받는다. 옴의 법칙이 전류 I_{ab} = V_a · G_{ab}를 준다. 이것은 영 데이터 전송으로 제자리에서 계산된 Hadamard 곱이다.

### 1.3 주장

이 대응은 우연이 아니다. Hadamard 곱은 다음을 만족하는 **유일한** 연산이다:
- 쌍선형(변분 안정성에 의해 요구)
- 점별(국소성에 의해 요구)
- 스칼라값(에지 수준 계산에 의해 요구)

이것들이 정확히 메모리 내 계산을 가능하게 하는 성질들이다. 행렬 곱은 쌍선형이지만 비국소적이다(각 출력이 전체 행과 열에 의존). 텐서 곱은 국소적이지만 스칼라값이 아니다. Hadamard 곱만이 세 가지 모두를 만족한다.

**메모리 내 처리는 공학적 선택이 아니다. 내용-구조 연산을 위한 유일한 아키텍처다.**

## 2. 네 가지 구조적 이점

### 2.1 자연스러운 Hadamard 계산

전도도 G_{ij}와 입력 전압 V_i를 가진 크로스바 배열에서:

I_{ij} = V_i × G_{ij}   (각 접합에서 옴의 법칙)

이것은 C → V, S → G인 T_{ij} = C_{ij} × S_{ij}다. 계산이 데이터의 물리적 위치에서 일어난다. 인출도, 전송도, 캐시도 없다.

크로스바에서의 표준 행렬-벡터 곱은 y_j = Σ_i V_i G_{ij}(키르히호프 전류 법칙이 열을 합산)를 계산한다. 이것은 Hadamard 곱이 **아니다** — 논문 A의 보조정리 1이 올바른 합성이 아님을 증명한 행렬 곱이다. 열 합산 전의 개별 접합 전류 I_{ij} = V_i G_{ij}가 Hadamard 곱이다.

### 2.2 자동 차수 정규화

자기수반 정규화 인접행렬 S = D⁻¹/²AD⁻¹/²(논문 A, 명제 2)는 각 에지를 1/√(d_source × d_target)로 스케일한다. 크로스바에서 이것은 다음에 사상된다:

G_{ij}^{normalized} = G_{ij} / √(Σ_k G_{ik}) / √(Σ_k G_{kj})

이것은 행과 열 단자의 저항성 전압 분배기에 의해 구현될 수 있으며, 총 행/열 전도도를 정규화 인자로 사용한다. 정규화는 대칭(자기수반)이다 — 각 에지가 소스만이 아닌 양 끝점에 의해 스케일된다.

이것은 차수 불균형을 자동 보정한다: 허브 노드(높은 차수)는 에지당 영향이 √d로 감소하여 각 이웃에게 동등한 발언권을 부여한다. 소프트웨어 수준 정규화가 불필요하다.

### 2.3 희소성에 자연스러운 복잡도

조밀 n×n 행렬에 대한 행렬 곱은 O(n²) 연산을 요구한다. |E|개 에지를 가진 희소 그래프에 대한 Hadamard 곱은 O(|E|) 연산을 요구한다 — 비영 항목만 계산된다.

크로스바에서 이것은 직접 사상된다: 비영 전도도(물리적 연결)를 가진 접합만 전류를 전달한다. 영 전도도 접합은 에너지를 소비하지 않는다. 하드웨어가 그래프 희소성을 자동으로 활용한다.

10⁶ 노드와 평균 차수 100의 소셜 네트워크: 행렬 곱은 10¹² 연산 필요; Hadamard 곱은 10⁸. 에너지와 시간에서 10,000배 차이.

### 2.4 변분 안정성이 비선형 PIM을 배제

변분 안정성 조건(논문 A, 정리 1)은 T = C^α S^β에서 α = β = 1을 요구한다. 배제되는 것:
- 이차 합성 T = C² · S (α = 2): 비균일 감도
- 메모리 내 활성화 T = σ(C ⊙ S), σ가 비선형: 선형성 위반
- α ≠ 1 또는 β ≠ 1인 멱법칙 합성: ∂²T/∂C² = 0에 의해 배제

하드웨어 용어로: 접합은 V² × G나 tanh(V × G)가 아닌 **선형** 곱 V × G를 계산해야 한다. 비선형 활성화 함수가 필요하면, 메모리 내 Hadamard 계산 **도중이** 아닌 **이후에** 적용해야 한다.

이것은 공학적 편의가 아닌 수학으로부터의 설계 제약이다.

## 3. 아키텍처

### 3.1 단층 PIM 그래프 프로세서

```
입력: 노드 특징 벡터 {v_i} (내용)
저장: 에지 전도도 {G_{ij}} (구조)
출력: 갱신된 노드 특징 {v'_i}

각 노드 i에 대해:
  1. 행 i에 전압 v_i 인가
  2. 접합 전류 읽기 I_{ij} = v_i × G_{ij}  (Hadamard 곱)
  3. 정규화: I_{ij}^{norm} = I_{ij} / √(d_i × d_j)  (차수 보정)
  4. 집계: v'_i = Σ_j I_{ij}^{norm}  (열 합, 키르히호프)
```

단계 1–2는 메모리 내(영 전송). 단계 3은 행/열 전도도 합 필요(사전 계산 가능). 단계 4는 물리적 합(키르히호프 전류 법칙).

### 3.2 다중 스케일 아키텍처

L개 층의 그래프 신경망:
- 직렬 L개 크로스바 배열
- 각 배열이 한 층의 에지 가중치 저장
- 층 l의 출력이 층 l+1에 직접 공급(중간 저장 없음)
- 총 데이터 전송: 입력 특징 입력 + 출력 특징 출력
- 모든 중간 계산: 메모리 내

### 3.3 기존 PIM과 비교

| 아키텍처 | 연산 | 데이터 전송 | 정규화 |
|---|---|---|---|
| 표준 GPU | 행렬 곱 | 층당 O(n²) | 소프트웨어(비대칭 가능) |
| 기존 PIM | 행렬-벡터 곱 | 층당 O(n) | 소프트웨어 |
| **Hadamard PIM** | **원소별 곱** | **층당 O(1)** | **하드웨어(대칭)** |

Hadamard PIM은 층당 데이터 전송을 완전히 제거한다. 유일한 전송은 초기 입력과 최종 출력이다.

## 4. 예상 성능

### 4.1 에너지

ReRAM 접합에서 연산당 에너지: ~1 fJ(펨토줄).
그래프 층당 연산: |E|(에지 수).
10⁶ 노드, 평균 차수 100의 그래프: |E| = 10⁸.
층당 에너지: 10⁸ × 10⁻¹⁵ J = 100 nJ.

GPU(같은 그래프에서 층당 약 10 μJ 추정) 대비: **100배 에너지 절감**.

### 4.2 지연

크로스바 계산은 아날로그이고 병렬: 모든 접합이 동시에 계산.
층당 지연: ~10 ns(크로스바 RC 시정수).
L = 3 층: 30 ns.

GPU(희소 행렬 연산에서 약 1 ms 추정) 대비: **30,000배 지연 감소**.

### 4.3 스케일링

에너지와 지연이 O(n²)가 아닌 O(|E|)로 스케일한다. 척도 없는 그래프(차수 분포가 멱법칙)에서 |E| ~ n × ⟨k⟩, 여기서 ⟨k⟩는 평균 차수(대부분의 실제 네트워크에서 상수). 따라서: **노드 수에 대한 선형 스케일링**.

## 6. 논의

### 6.1 이것이 무엇인가

이 논문은 메모리 내 처리가 단순히 더 빠르거나 효율적인 것이 아니라 — T = C⊙S를 계산하기 위한 **구조적으로 올바른** 아키텍처임을 확립한다. Hadamard 곱은 국소성을 요구하고; 국소성은 저장과 계산의 동일 위치를 요구하며; 동일 위치의 저장과 계산이 메모리 내 처리의 정의다.

### 6.2 이것이 무엇이 아닌가

이 논문은 모든 계산이 메모리 내에서 이루어져야 한다고 주장하지 않는다. Hadamard 곱이 아닌 연산(예: 정렬, 분기, 순차 논리)은 이 아키텍처의 혜택을 받지 않는다. 주장은 구체적이다: 그래프 위의 내용-구조 연산에 대해 PIM이 유일한 올바른 아키텍처다.

### 6.3 시리즈와의 관계

논문 A가 T = C⊙S의 유일성을 증명했다. 논문 C가 GNN 벤치마크에서 대안을 능가함을 보였다. 이 논문은 그 하드웨어 실현이 메모리 내 처리임을 보인다. 수학적 구조가 계산을 결정하고; 계산이 아키텍처를 결정하며; 아키텍처가 하드웨어를 결정한다.

## 7. 결론

Hadamard 곱 T = C⊙S는 유일한 내용-구조 합성이다(논문 A). 그것은 점별적이고, 쌍선형이며, 스칼라다 — 메모리 내 계산을 가능하고 올바르게 만드는 세 성질. ReRAM 크로스바는 옴의 법칙을 통해 이 곱을 자연스럽게 계산하며, 전도도 스케일링을 통한 자동 차수 정규화, 희소성 활용을 통한 O(|E|) 복잡도, 물리적 동일 위치를 통한 영 데이터 전송을 갖는다. 메모리 내 처리는 최적화가 아니다 — 필연적 연산을 위한 필연적 아키텍처다.

## 감사의 글

저자는 수학적 검증, 수치 시뮬레이션, 원고 작성 보조를 위한 계산 도구로 Claude (Anthropic)를 사용하였다. 공리, 정리, 증명, 물리적 해석, 전략적 판단을 포함한 모든 이론적 내용은 저자의 독창적 작업이다. 저자가 모든 내용에 대해 전적인 책임을 진다.
